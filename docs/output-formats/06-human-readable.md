# Форматы для людей и ревью

Документы и отчёты для совместной работы, ревью, передачи аналитикам и архивирования.

---

## Markdown-отчёт — `overview.md`

**Назначение:** главный обзорный документ. Читается в GitHub, VS Code, Obsidian без дополнительных инструментов.

**Структура:**
```markdown
# Анализ запроса: 258_npo_analysis

## Общая информация
- Всего узлов: 296
- Временных таблиц (ВТ): N
- Подзапросов: M
- Источников: K

## Список ВТ
| ID | Имя | Тип | Источники | Дети |
|---|---|---|---|---|

## Граф зависимостей
[Диаграмма Mermaid]

## Ключевые поля
### НомерДоговора
[Цепочка lineage]

## Проблемные места
- Узел N: сложное вычисление без комментариев
```

## HTML-отчёт — `overview.html`

**Назначение:** наглядный просмотр в браузере с поиском, фильтрацией, раскрывающимися блоками.

**Ключевые элементы:**
- Интерактивная таблица ВТ с фильтрацией
- Встроенный Cytoscape.js или D3.js граф
- Раскрывающийся текст запроса по каждой ВТ
- Highlight ключевых полей

---

## PDF-отчёт — `overview.pdf`

**Назначение:** передача вовне, архив, согласование с заказчиком.

**Генерация:** из HTML через `wkhtmltopdf` или headless Chrome, из Markdown через Pandoc.

```bash
pandoc overview.md -o overview.pdf --pdf-engine=xelatex
```

## XLSX — `catalog.xlsx`

**Назначение:** сводный каталог для Excel. Несколько листов в одном файле — самый удобный формат для ручной аналитики.

**Листы:**
| Лист | Содержимое |
|---|---|
| `Nodes` | Все узлы с метаданными |
| `TempQueries` | Только ВТ с каталогом |
| `Edges` | Связи родитель-потомок и ссылки |
| `Sources` | Карта источников |
| `Fields` | Реестр полей |
| `Lineage` | Lineage ключевых полей |
| `Joins` | Карта соединений |
| `Conditions` | Карта условий |

**Генерация (Python):**
```python
with pd.ExcelWriter('catalog.xlsx') as writer:
    nodes_df.to_excel(writer, sheet_name='Nodes', index=False)
    tq_df.to_excel(writer, sheet_name='TempQueries', index=False)
```

## README каталога — `README.md` (в папке артефактов)

**Назначение:** объяснить другому разработчику (или себе через месяц) что находится в наборе файлов.

**Должен содержать:**
- Источник данных (какой запрос, откуда)
- Дата и версия генерации
- Краткое описание каждого файла
- Порядок работы с набором
