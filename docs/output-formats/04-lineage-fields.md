# Форматы lineage полей и ключевых цепочек

Файлы для трассировки происхождения полей: откуда берётся каждое значение, через какие ВТ проходит, где формируется.

---

## Поля узлов — `fields.csv`

**Назначение:** реестр всех выходных полей по каждому узлу. Основа для построения lineage.

**Структура:**
```csv
node_id,node_name,field_ordinal,field_alias,expression,source_table,source_field,is_computed
132,ВТ_ПенсионныеСчета,1,НомерДоговора,ПС.НомерДоговора,ПС,НомерДоговора,false
132,ВТ_ПенсионныеСчета,2,ВидОбязательств,ПС.ВидОбязательств,ПС,ВидОбязательств,false
138,ВТ_Расчёт,3,СуммаВзносов,"ISNULL(Д.Сумма, 0)",Д,Сумма,true
```

**Поля:**
| Поле | Описание |
|---|---|
| `field_alias` | Псевдоним поля в выборке |
| `expression` | Полное выражение (`ISNULL(...)`, `СЛУЧАЙ КОГДА ...`) |
| `source_table` | Алиас исходной таблицы |
| `source_field` | Физическое поле источника |
| `is_computed` | Вычисляемое выражение (не простая ссылка) |

---

## Lineage полей — `field_lineage.json`

**Назначение:** главный файл для понимания происхождения каждого поля. Цепочка от финального поля к физическому источнику через все ВТ.

**Структура:**
```json
[
  {
    "output_field": "НомерДоговора",
    "final_node_id": 293,
    "chain": [
      {"node_id": 293, "node_name": "ВТ_Итог", "alias": "НомерДоговора", "expr": "ПС.НомерДоговора"},
      {"node_id": 132, "node_name": "ВТ_ПенсионныеСчета", "alias": "НомерДоговора", "expr": "ПС.НомерДоговора"},
      {"node_id": null, "node_name": null, "source_table": "РегистрНакопления.ПенсионныеСчета", "source_field": "НомерДоговора"}
    ],
    "depth": 3
  }
]
```

---

## Lineage ключевых полей — `lineage_key_fields.json`

**Назначение:** готовые цепочки для ключевых граф отчёта. Позволяет сразу видеть происхождение самых важных полей.

**Ключевые поля для НПО:**
- `НомерДоговора`
- `ВидОбязательств`
- `ПенсионныйСчет`
- `КодСхемы`
- `СостояниеСчёта`

**Структура:**
```json
{
  "generated_at": "2026-06-02T12:00:00",
  "source_file": "258_npo_analysis.json",
  "key_fields": [
    {
      "field": "НомерДоговора",
      "description": "Уникальный ключ пенсионного договора",
      "chain": [...],
      "branching": false,
      "notes": ""
    },
    {
      "field": "ВидОбязательств",
      "description": "Классификатор типа обязательства: ОПС, НПО, ПК",
      "chain": [...],
      "branching": true,
      "notes": "Разветвление по транзитным счетам в узле 138"
    }
  ]
}
```

## Реестр вычисляемых выражений — `expressions.csv`

**Назначение:** выделить все сложные вычисления (ВЫБОР КОГДА, ISNULL, ПОДСТРОКА, агрегаты) для отдельного анализа.

**Структура:**
```csv
node_id,node_name,expr_type,expr_text,output_alias,line
138,ВТ_Расчёт,ISNULL,"ISNULL(Д.Сумма, 0)",СуммаВзносов,12
293,ВТ_Итог,CASE,"СЛУЧАЙ КОГДА ПС.ВидОбязательств = &ОПС ТОГДА 1 ИНАЧЕ 0 КОНЕЦ",ПризнакОПС,7
```

**Типы выражений:** `ISNULL`, `CASE`, `AGGREGATE`, `SUBSTRING`, `CAST`, `FUNCTION`, `ARITHMETIC`, `PARAMETER`

---

## Карта условий — `conditions.csv`

**Назначение:** анализ логики отбора данных: WHERE, условия JOIN, HAVING.

**Структура:**
```csv
node_id,node_name,condition_type,expression,involved_tables,involves_parameter
132,ВТ_ПенсионныеСчета,WHERE,ПС.Период МЕЖДУ :НачДата И :КонДата,ПС,true
138,ВТ_Расчёт,JOIN_ON,Д.НомерДоговора = ПС.НомерДоговора,"Д;ПС",false
```

---

## Join map — `joins.csv`

**Назначение:** карта всех соединений таблиц. Помогает понять, через какие поля связаны данные.

**Структура:**
```csv
node_id,node_name,join_type,left_source,left_alias,right_source,right_alias,on_expression
138,ВТ_Расчёт,LEFT JOIN,ВТ_ПенсионныеСчета,ПС,ВТ_Договоры,Д,ПС.НомерДоговора = Д.НомерДоговора
```

**Типы JOIN:** `INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`, `CROSS JOIN`

---

## Mapping полей — `field_mapping.csv`

**Назначение:** прямое сопоставление выходных полей отчёта с исходными физическими полями. Удобен для проверки полноты покрытия.

**Структура:**
```csv
output_field,final_node_id,intermediate_node_id,input_field,rule_type,transform_desc
НомерДоговора,293,132,НомерДоговора,direct,Прямой перенос
ВидОбязательств,293,132,ВидОбязательств,conditional,ISNULL с заменой пустого
```

---

## Матрица зависимостей — `dependency_matrix.csv`

**Назначение:** быстрая проверка: зависит ли узел X от узла Y. Полезна для анализа влияния изменений.

**Структура:** строки — зависимые узлы, столбцы — исходные узлы, значение — `1` (зависит) / `0` (не зависит).

```csv
node_id,132,138,293
293,1,1,0
138,1,0,0
```
