# Форматы текста и декомпиляции

Файлы для работы с исходными текстами узлов запроса: разбивка на файлы, индексирование, нормализация.

---

## Файлы текстов по узлам — `query_texts/node_{id}.sql`

**Назначение:** читать и ревьюить текст одного узла отдельно, без контекста всего JSON.

**Структура:**
```sql
-- Node ID: 132
-- Name: ВТ_ПенсионныеСчета
-- Type: tempquery
-- Parent: null
-- OwnInTables: РегистрНакопления.ПенсионныеСчета

ВЫБРАТЬ
    ПС.НомерДоговора,
    ПС.Контрагент,
    ПС.ВидОбязательств
ИЗ
    РегистрНакопления.ПенсионныеСчета КАК ПС
```

**Именование:** `node_{id}.sql` или `node_{id}_{name}.sql`

---

## Markdown-файлы по узлам — `query_texts/node_{id}.md`

**Назначение:** документирование узла с метаданными, комментариями и аннотациями. Удобно для совместной работы.

**Структура:**
```markdown
# ВТ_ПенсионныеСчета (id: 132)

**Тип:** tempquery  
**Родитель:** нет (корневая ВТ)  
**Дети:** [133]  
**Источники:** РегистрНакопления.ПенсионныеСчета  

## Текст запроса

```sql
ВЫБРАТЬ ...
```

## Аннотации

<!-- Поле НомерДоговора: уникальный ключ пенсионного договора -->
```

## Индекс текстов — `texts_index.json`

**Назначение:** каталог всех текстовых файлов с хешами для контроля изменений и навигации.

**Структура:**
```json
[
  {
    "node_id": 132,
    "name": "ВТ_ПенсионныеСчета",
    "path": "query_texts/node_132.sql",
    "text_len": 1240,
    "text_hash": "sha256:a1b2c3...",
    "line_count": 48
  }
]
```

---

## Нормализованный SQL — `normalized_queries.sql`

**Назначение:** канонизированный текст всех ВТ для сравнения логики без косметических различий (пробелы, регистр, псевдонимы).

**Правила нормализации:**
- Привести ключевые слова к верхнему регистру
- Удалить лишние пробелы и переносы строк
- Нормализовать псевдонимы таблиц
- Добавить разделитель между ВТ: `-- ===== ВТ_Имя (id=N) =====`

## Токенизированный CSV — `tokens.csv`

**Назначение:** лексический анализ, поиск паттернов (ISNULL, CASE, конкретные поля), статистика по использованию.

**Структура:**
```csv
node_id,token,token_type,position,line
132,ВЫБРАТЬ,KEYWORD,0,1
132,НомерДоговора,FIELD,8,2
132,ISNULL,FUNCTION,45,8
```

**Типы токенов:** `KEYWORD`, `FIELD`, `TABLE`, `ALIAS`, `FUNCTION`, `LITERAL`, `OPERATOR`, `COMMENT`

---

## AST-подобный JSON — `ast_nodes.json`

**Назначение:** машинная обработка структуры запроса без парсинга сырого текста. Строится на основе `text` после лексического и синтаксического анализа.

**Структура:**
```json
{
  "node_id": 132,
  "select": [
    {"alias": "НомерДоговора", "expr": "ПС.НомерДоговора", "source": "ПС"},
    {"alias": "ВидОбязательств", "expr": "ПС.ВидОбязательств", "source": "ПС"}
  ],
  "from": [
    {"table": "РегистрНакопления.ПенсионныеСчета", "alias": "ПС"}
  ],
  "where": "ПС.Период МЕЖДУ :НачалоПериода И :КонецПериода",
  "joins": [],
  "group_by": []
}
```

## Комментарии отдельно — `comments.csv`

**Назначение:** вынести служебные ремарки, исторические комментарии, аннотации к полям для отдельного анализа.

**Структура:**
```csv
node_id,comment_text,position,line,comment_type
132,// Добавлено в рамках задачи IFRS 17,0,1,line
```

---

## Diff-friendly text — `node_{id}.norm.txt`

**Назначение:** версионирование запросов и diff между версиями. Используется в git-сравнениях.

**Правила:** одно выражение/колонка на строку, стандартные отступы, без автогенерированных комментариев.
