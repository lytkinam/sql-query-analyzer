# Форматы для поиска, индексации и AI-обработки

Файлы для семантического поиска, RAG-пайплайнов, LLM-обработки и векторных баз.

---

## Инвертированный индекс — `inverted_index.json`

**Назначение:** быстрый поиск узлов по токенам (именам полей, таблиц, функциям).

**Структура:**
```json
{
  "НомерДоговора": [132, 138, 293, 294, 295],
  "ISNULL": [138, 214, 276],
  "РегистрНакопления.ПенсионныеСчета": [132]
}
```

**Использование:** `index["НомерДоговора"]` → список всех узлов, где встречается поле.

---

## Search corpus JSONL — `corpus.jsonl`

**Назначение:** подготовка к семантическому поиску (RAG, ElasticSearch, OpenSearch). Каждая строка — один документ.

**Структура:**
```jsonl
{"id": "node_132", "title": "ВТ_ПенсионныеСчета", "body": "ВЫБРАТЬ НомерДоговора, ВидОбязательств...", "tags": ["tempquery", "РегистрНакопления"], "path": "query_texts/node_132.sql", "node_id": 132, "type": "tempquery"}
```

**Поля:**
| Поле | Описание |
|---|---|
| `id` | Уникальный строковый идентификатор |
| `title` | Имя ВТ или подзапроса |
| `body` | Полный текст узла |
| `tags` | Теги: тип, источники, ключевые поля |
| `path` | Путь к файлу с текстом |

## Chunks JSONL — `chunks.jsonl`

**Назначение:** нарезка больших текстов узлов на части для LLM с ограничением контекста. Каждый чанк — независимый фрагмент.

**Структура:**
```jsonl
{"chunk_id": "node_132_0", "node_id": 132, "chunk_index": 0, "total_chunks": 3, "text": "ВЫБРАТЬ\n    ПС.НомерДоговора,\n    ПС.ВидОбязательств", "char_start": 0, "char_end": 512}
{"chunk_id": "node_132_1", "node_id": 132, "chunk_index": 1, "total_chunks": 3, "text": "ИЗ\n    РегистрНакопления.ПенсионныеСчета КАК ПС", "char_start": 513, "char_end": 1024}
```

**Параметры нарезки:** рекомендуемый размер чанка — 512–1024 токена, перекрытие — 10–20%.

## Embeddings Parquet — `embeddings.parquet`

**Назначение:** хранение векторных представлений текстов узлов для семантического поиска и кластеризации похожих ВТ.

**Структура (колонки):**
`chunk_id, node_id, chunk_index, vector (list[float]), text, model_name, embedding_dim`

**Генерация (Python):**
```python
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('intfloat/multilingual-e5-large')
vectors = model.encode(texts)
```

## Keyword map CSV — `keywords.csv`

**Назначение:** тематическая разметка узлов по ключевым терминам. Помогает группировать ВТ по смысловым кластерам.

**Структура:**
```csv
node_id,node_name,keyword,weight,category
132,ВТ_ПенсионныеСчета,НомерДоговора,0.95,key_field
132,ВТ_ПенсионныеСчета,ВидОбязательств,0.90,key_field
132,ВТ_ПенсионныеСчета,IFRS17,0.75,domain
```

## Alias dictionary JSON — `aliases.json`

**Назначение:** нормализация имён полей и ВТ для единообразного поиска.

**Структура:**
```json
{
  "НомерДоговора": ["НомДоговора", "Договор.Номер", "ДокументОснование.Номер"],
  "ВидОбязательств": ["ВидПенсии", "ТипОбязательства"],
  "ВТ_ПенсионныеСчета": ["ВТ_ПС", "ВТ_Счета"]
}
```
